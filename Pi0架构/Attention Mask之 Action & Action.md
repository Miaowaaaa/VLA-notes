# Attention Mask之 Action & Action

> ## **问题：自动驾驶中经常把决策规划建模成Markov过程，为什么这里允许Action之间可以互相看到。**


\
**在经典马尔可夫假设中**确实是 `P(s_{t+1} | s_t, a_t)`，下一状态只依赖当前状态。但这个假设在**驾驶动作生成**里是有问题的，原因如下：

---

## 1. 马尔可夫假设本身在驾驶中就不成立

经典 MDP 假设是**数学简化**，不是物理现实。

**真实驾驶场景**：

```
你看到前方 200 米处有个急弯
         ↓
你现在就开始减速（而不是到了弯口才刹）
         ↓
你的当前动作（减速）取决于"未来的路况"
```

如果严格遵循马尔可夫假设，驾驶员应该**只根据当前瞬时状态**（车速 60，方向盘 0 度）决定动作。但现实中司机会**前瞻（lookahead）**很远，根据未来道路曲率提前调整。

> **驾驶是"预测未来 → 规划现在"的过程，不是"只看现在 → 反应式输出"。**

---

## 2. 双向注意力的本质：学的是"物理约束"，不是"预测未来"

你可能担心的点是：**frame_3 看到 frame_5，是不是在偷看未来答案？**

关键区分：

| | 自回归文本生成 | 动作序列预测 |
|---|---|---|
| **目标** | 逐个预测下一个 token | 一次性生成整条轨迹 |
| **frame_5 是什么** | 尚未生成的未知答案 | **已知的物理约束条件** |
| **双向注意力的作用** | 作弊（看到未生成的答案） | 学"整条轨迹应该满足什么约束" |

### 具体例子：轨迹平滑性约束

假设真实轨迹是：

```
frame_1: 转向 0.0
frame_2: 转向 0.1    ← 开始入弯
frame_3: 转向 0.3
frame_4: 转向 0.5    ← 弯心
frame_5: 转向 0.3    ← 开始出弯
frame_6: 转向 0.1
frame_7: 转向 0.0
frame_8: 转向 -0.1
```

**物理约束**：
- 转向角不能突变：`|θ_{t+1} - θ_t|` 不能太大（受限于车辆转向角速度）
- 轨迹应该是**连续可微**的（加速度有限）

如果 frame_3 只能看 frame_1~2（单向/因果），它**不知道后面要出弯**，可能会输出一个过大的转向角，导致 frame_4~5 无法平滑过渡。

如果 frame_3 能看到 frame_5~7（双向），它会知道"后面要出弯"，于是 frame_3 的转向角会更合理，整条轨迹自然满足**曲率连续性**。

> **双向注意力不是让 frame_3 "抄袭" frame_5 的答案，而是让 frame_3 "参考" frame_5 的约束，输出一个更符合物理规律的值。**

---

## 3. 类比：最优控制（MPC）也是看整个 Horizon

在自动驾驶的经典算法里，**模型预测控制（MPC）** 的优化目标长这样：

\[
\min_{u_0, u_1, ..., u_{N-1}} \sum_{t=0}^{N-1} \|x_t - x_{ref}\|^2_Q + \|u_t\|^2_R + \|u_{t+1} - u_t\|^2_{\Delta}
\]

注意到：
- **优化变量是整条控制序列** `u_0` 到 `u_{N-1}`，不是只优化 `u_t`
- **代价函数包含差分项** `u_{t+1} - u_t`，惩罚控制量的突变
- 求解时**所有时间步同时考虑**，因为 `u_t` 和 `u_{t+1}` 的平滑项把它们耦合在了一起

**MPC 在优化 `u_t` 时，必须同时知道 `u_{t-1}` 和 `u_{t+1}`**，否则无法保证平滑性。

`Emu3Pi0` 的双向注意力本质上在做同样的事：**像 MPC 一样，整条轨迹一次性优化，帧间互相约束。**

---

## 4. 另一个类比：样条插值

假设你要用 8 个控制点拟合一条 B 样条曲线：

```
p_1, p_2, p_3, p_4, p_5, p_6, p_7, p_8
```

B 样条的数学定义是：

\[
S(t) = \sum_{i=1}^{8} B_{i,k}(t) \cdot p_i
\]

其中 **基函数 `B_{i,k}` 是非局部的**——每个控制点 `p_i` 会影响整条曲线的多个区间。

如果你只允许 `p_4` 参考 `p_1~p_3`（因果/单向），你就**无法拟合出一条平滑的曲线**，因为 B 样条的本质就是全局基函数的加权。

**Flow Matching 的动作生成和 B 样条/神经网络隐式场类似**：输出不是独立的点，而是**受全局约束耦合的连续场**。双向注意力就是这个耦合机制。

---

## 5. 那为什么 State（历史）用单向、Action（未来）用双向？

| 模块 | 方向 | 原因 |
|------|------|------|
| **State token**（历史状态） | 单向（隐含在序列顺序中） | 历史是已经发生的，确实有因果顺序 |
| **Action frames**（未来动作） | 双向 | 未来是同时规划的，帧间是物理约束关系 |

**关键区别**：
- **历史** = 已经观测到的轨迹，必须按时间顺序处理
- **未来** = 尚未执行但**同时决策**的规划轨迹，需要全局优化

---

## 一句话总结

> **马尔可夫假设适用于"状态转移"（物理世界如何演化），但不适用于"轨迹规划"（如何生成一条满足约束的未来路径）。双向注意力让模型像 MPC 或样条拟合一样，在生成整条轨迹时满足平滑性、连续性等全局物理约束，而不是孤立地逐帧预测。**
